## My process

### Built with

- Semantic HTML5 markup
- CSS custom properties
- Flexbox layout
- Mobile-first workflow
- Google Fonts (Figtree family)
- CSS transitions for interactive states

### What I learned

- How to structure a card component using semantic HTML and BEM methodology for class naming
- Implementing hover states and CSS transitions for better user interaction
- Using Flexbox to create responsive layouts and align content effectively
- Box shadow and border-radius for creating depth and modern card designs
- How to work with Google Fonts and custom font families in CSS

```css
.card__body-title:hover {
  color: hsl(47, 88%, 63%);
  cursor: pointer;
  transition: 0.3s ease-in-out;
}
```

This project reinforced my understanding of component-based design and the importance of semantic HTML for accessibility.

### Continued development

- Implement responsive design to work across different screen sizes (tablet and large desktop views)
- Explore CSS Grid for more complex layouts
- Add JavaScript interactivity for more dynamic card behaviors
- Practice accessibility improvements (ARIA labels, keyboard navigation)
